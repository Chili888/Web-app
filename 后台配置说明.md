# 国内精品肉铺后台配置说明

正式 Mini App 地址：

```text
https://bot.cverseintl.cloud/
```

后台地址：

```text
https://bot.cverseintl.cloud/admin/
```

GitHub Pages 地址仅在服务器前端需要回滚时临时使用。

## 一、创建 Supabase 项目

1. 登录 Supabase。
2. 创建一个新项目。
3. 等待项目初始化完成。

## 二、初始化数据库

1. 打开 Supabase Dashboard。
2. 进入 **SQL Editor**。
3. 打开本仓库中的 `supabase-setup.sql`。
4. 复制全部内容到 SQL Editor。
5. 点击 **Run**。

脚本会创建：

- `products`：商品资料
- `store_settings`：店铺资料
- `admin_profiles`：管理员白名单
- `product-images`：商品图片存储桶
- RLS 数据权限策略

## 三、创建管理员账号

1. 进入 **Authentication → Users**。
2. 点击 **Add user**。
3. 填写管理员邮箱和密码。
4. 勾选自动确认邮箱（Auto Confirm User）。

然后在 SQL Editor 单独运行：

```sql
insert into public.admin_profiles (user_id, display_name)
select id, '店铺管理员'
from auth.users
where email = '替换成你的管理员邮箱'
on conflict (user_id) do update
set display_name = excluded.display_name;
```

## 四、连接网站和 Supabase

打开 Supabase 项目的 **Project Settings → API**，找到：

- Project URL
- anon key 或 publishable key

编辑仓库根目录的 `config.js`：

```js
window.APP_CONFIG = Object.freeze({
  supabaseUrl: "你的 Project URL",
  supabaseAnonKey: "你的 anon/publishable key",
  pageUrl: "https://bot.cverseintl.cloud/"
});
```

这里只能填写 anon/publishable key。

**绝对不要把 `service_role` 密钥放进 GitHub 或网页。**

## 五、登录后台

服务器前端发布后，打开：

```text
https://bot.cverseintl.cloud/admin/
```

登录后可以：

- 新增商品
- 编辑商品
- 删除商品
- 上传商品图片
- 设置价格、库存和排序
- 上架或下架商品
- 修改店铺名称
- 修改客服、频道和群组账号
- 修改价格、库存、配送、售后及防骗说明

保存后，前台重新打开或刷新即可看到最新内容。

## 六、日常使用

以后不需要进入 GitHub 修改商品。

正常流程：

```text
打开后台
→ 登录
→ 新增或编辑商品
→ 保存
→ 前台更新
```

`config.js` 和数据库配置完成后，商品与页面文案的日常更新不需要重建前端镜像。
